# Rules

Tier 3 of `mur check`'s did-you-mean engine: hand-authored AST-shape rules that catch idiomatic mistakes Tier 2 (Roslyn fuzzy match) cannot reach because the *fix* is a tree rewrite, not a rename. Each rule lives in its own file as a class implementing `IRulePattern`, ships with a unit test against captured `(broken, fixed)` pairs from spec 037's `fixes.jsonl` corpus, and carries a kill-switch (`mur check --disable-rule <Name>`). See [spec 038 §6](../../../../docs/specs/038-mur-check-did-you-mean-design.md#6-tier-3--induced-pattern-rules) for the rule contract and [the implementation tasks' Validation Gate](../../../../docs/specs/tasks/038-mur-check-did-you-mean-implementation.md#human-validation-gate-the-dont-codify-bad-rules-mechanism) for the six bars every rule must clear before merge.
