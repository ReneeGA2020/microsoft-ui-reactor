## What surprised us

1. **Two TIER_004 satisfaction patterns shipped in parallel.** Spec
   §11 originally read "≥1 `screenshot://` reference resolved" — which
   assumes a runnable doc app that produces a capture. Under-the-hood
   pages don't have doc apps; their visual asset is a Mermaid diagram
   that renders to SVG into `docs/guide/images/<topic>/<id>.svg`. Wave
   A relaxed `REACTOR_DOC_TIER_004` in `TierLint.cs` to accept either a
   resolved `screenshot://` reference OR an inline `images/<topic>/`
   diagram reference (commit `2ecee29`). Wave C, working in parallel,
   instead created minimal `docs/_pipeline/apps/<topic>/` shells with
   `_placeholder.cs` + `doc-manifest.yaml` declaring the Mermaid output
   as `format: svg` so it counted as a screenshot. Both patterns passed
   the lint; the two approaches were committed within ~90 minutes of
   each other. **Resolution: keep wave-A's relaxation, drop wave-C's
   shells.** The shells are doc rot — empty C# files with one-line
   namespaces, exist solely to placate `CompileCommand.DiscoverApps`
   (which discovers apps by presence of a `.cs` file), and conflict
   with the design intent that doc-pipeline apps are runnable artifacts
   that capture screenshots. The lint relaxation reflects the actual
   semantics: a Solid+ page needs ≥1 visual reference, full stop. The
   review commit deletes the five shells and converts the five
   wave-C templates from `screenshot://<topic>/<id>` to
   `images/<topic>/<id>.svg`. Spec text should follow on Phase 4 to
   match.

2. **The "promotes legacy reference file" front-matter pattern works.**
   `reconciliation.md.dt` and `hooks-internals.md.dt` both declare in
   their goal block that they absorb `docs/reference/reconciliation.md`
   and `docs/reference/state-and-hooks.md` respectively. Wave A
   deleted the legacy files in the same commits. The grep across the
   repo found no live inbound links — every remaining hit is in spec
   docs (021, 041) describing the migration intent, in the task list
   marking the migration done, or in two stale stub pages
   (`docs/guide/reconciliation.md`, `docs/guide/hooks-internals.md`)
   that were generated by an older pipeline run pre-Phase-3.5 and will
   regenerate on the next full `mur docs compile`.

3. **Snippet `source:` resolution survives parallel waves.** The
   setup commit landed 90+ `#region <id>...#endregion` markers across
   five source areas (Core, Animation, Hooks, Hosting, Diagnostics);
   the three waves consumed ~50 unique markers across 15 pages.
   Validation found zero `REACTOR_DOC_SNIPPET_*` findings on first
   pass after the merge — the region IDs all match. No wave needed
   to add markers their wave didn't already own, which is a good
   signal that the setup commit cataloged the right surface.

4. **Some markers are reused across waves.** `RenderContext.cs#use-state-slot`
   resolves on six different templates (architecture-overview,
   reactivity-model, hooks-internals, effects-scheduling, threading-and-dispatch,
   modifier-system). `Component.cs#render-loop` appears on five.
   `Reconciler.cs#reconciler-entry` on three. The pipeline doesn't
   dedupe — same snippet, multiple inclusions — and that's
   correct: each page wants the same source excerpt in its own
   context. Worth knowing: if we shorten a marker region in source,
   the change propagates across every dependent page on next compile.

5. **`Modify`'s generic-T discipline is real and load-bearing.** The
   modifier-system page documents the trick that lets
   `.FontSize(24).Bold().Margin(8)` stay typed as the concrete element
   (`TextBlockElement`) all the way through the chain rather than
   collapsing to `Element`. Wave B's first draft glossed it; the
   second pass surfaced it as the structural difference vs. SwiftUI's
   `some View` / Compose's `Modifier`. This is the kind of design
   point the under-the-hood pages exist for — readers shouldn't have
   to discover it by reading the extension-method signatures.

6. **The element-pool state machine isn't documented anywhere else.**
   `element-pool.md.dt` walks the unpooled → Rented → Returning →
   Pooled cycle with the per-type-cap and compositor-taint exits.
   Wave B's source pass surfaced the "compositor-taint" condition
   (a poolable control that picked up a Composition animation can't
   safely be reused because the composition handle is still
   referenced) — that exit is in `ElementPool.cs#pool-return` and
   wasn't called out in any other page. Net new internals knowledge.

7. **The threading-and-dispatch trampoline name finally lands.**
   `ThreadAffinity.cs#dispatcher-trampoline` is the four-line dispatch
   guard that intercepts cross-thread setter calls. Wave B's page
   uses "trampoline" consistently; spec 041 had been using "marshal"
   in a few places. Phase 4 cross-link sweep will pick a vocabulary.

## Process notes

- **Three parallel waves stayed disciplined on the Reactor.csproj
  shared-build rule.** None of the wave agents ran `dotnet build` or
  `dotnet test` against `Reactor.csproj` concurrently. Validation runs
  through `dotnet run --project src/Reactor.Cli` which doesn't touch
  the XAML compiler's `input.json`. The waves landed 5 + 5 + 5 commits
  in approximately 3 hours without a single rebuild race.
- **The setup commit (`dc5ed6b`) was load-bearing.** It pre-cataloged
  every `#region` marker the three waves would later cite. A single
  upfront pass through `src/Reactor/Core/`, `Animation/`, `Hooks/`,
  `Hosting/`, `Diagnostics/`, and `Reactor.Analyzers/` cost ~20 minutes
  and saved each wave from having to add markers ad-hoc against a
  hot-build target. Repeat this pattern for any future content phase
  that pulls heavily from source.
- **TIER_004 normalization caught only after the fact.** Both patterns
  passed individual validation in their respective waves; the
  consolidation review caught the inconsistency by inspecting `git
  status` and seeing both `_pipeline/apps/<topic>/` directories *and*
  `TierLint.cs` modifications in the merged diff. Lesson: a brief
  cross-wave coordination message ("if you hit TIER_004, here's the
  pattern we picked") would have avoided the duplicated solution
  surface. Worth doing for Phase 4 if it runs parallel waves.
- **The mental-model lead survives parallel authoring.** Every Phase
  3.5 Comprehensive page opens with 100+ words of mental-model prose
  above the first `#`-heading. Voice is consistent across waves — no
  page reads like a different author's hand. Crediting that to the
  AI Author Skill's anti-slop checklist plus the existing Phase 1–3
  page set as a voice reference.
- **One vocabulary drift surfaced.** "Trampoline" vs. "marshal" for
  the cross-thread setter dispatch guard — wave B picked
  "trampoline" consistently (it's the existing source-region name);
  spec 041 §15 has one stray "marshal". Phase 4 sweep.
